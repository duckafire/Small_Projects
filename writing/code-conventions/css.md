### CSS

###### Last changes: 07 March 2026

* `{` must always go on the next line after their *owner*.

* `}` must always go on the next line after the *block body*.

* Always prefer using classes.

* Always avoid creating complex/strict selectors.

* *Comma-separated lists of selectors* must always be listed vertically.

* Use *comma-separated lists of selectors*, or *subclasses* to avoid much duplicated code.

* Avoid aligning horizontally long *comma-separated* values of rules; like the values of
  `box-shadow` and `transition`.

* Always put a semicolon after the last style rule of a *block*.
